# 严格原地旋转模式 - 快速参考

**版本**: V1.0
**日期**: 2026-04-03
**状态**: ✅ 已实现并测试

---

## 核心改进

**问题**：原有的原地旋转功能不够严格，当角度在30-90度之间时，机器人仍可直行。

**解决**：实现严格的原地旋转状态机 - **一旦进入（>90°），必须转到<30°才能退出**。

---

## 行为说明

### 进入条件
- 角度误差 **> 90°** 且当前不在旋转模式
- 进入后速度强制为 0.0 m/s
- 只允许原地旋转，禁止直行

### 退出条件
- **必须**已在旋转模式 **且** 角度误差 **< 30°**
- 两个条件缺一不可
- 退出后恢复正常行驶

### 关键特性
✅ **严格性**：在旋转期间（30-90°），速度始终保持0
✅ **安全性**：每次控制循环都强制锁定速度
✅ **可观测性**：每次循环都输出详细日志

## 常见问题

**Q: 为什么选择90°作为进入阈值？**
A: 90°是典型的"大角度转弯"场景，此时机器人需要完全调整朝向。

**Q: 为什么选择30°作为退出阈值？**
A: 30°确保机器人已经基本对齐路径，可以安全开始直行。

**Q: 如果机器人在旋转期间收到新目标怎么办？**
A: 旋转状态由当前角度误差决定，新目标会重新计算角度误差。

**Q: 旋转模式下机器人会倒退吗？**
A: 不会。速度强制为0，只有角速度可以非零，确保纯原地旋转。
