# 原地旋转MPC失败回退机制修复报告

**日期**: 2026-04-05
**问题**: 机器人"转一下停一下"（间歇性旋转）
**状态**: ✅ **已修复**

---

## 🔍 **问题描述**

### **症状**

机器人在原地旋转时出现间歇性停止：
- ✅ 旋转模式能正确进入（角度>90°）
- ✅ 角速度能正确覆盖为固定值（0.5 rad/s）
- ❌ **但旋转会间歇性停止** - "转一下停一下"
- ❌ 不是平滑的连续旋转

### **诊断发现**

通过日志分析发现：

```
🔄 [PURE ROTATION] Speed LOCKED at 0.0 m/s | Angular vel: -0.5 rad/s
MPC solve failed with status: -11  ← MPC失败！
Publishing zero velocity...        ← 发布零速度！ ← 问题根源！
🔄 [PURE ROTATION] Speed LOCKED at 0.0 m/s | Angular vel: -0.5 rad/s
MPC solved successfully             ← MPC成功
```

**关键发现**：
- MPC求解在旋转期间**频繁失败**（状态码：-11, -3, -2, 4）
- 每次失败后发布**零速度**（v=0, ω=0）
- 这导致旋转被频繁打断 → "转一下停一下"

---

## 🔬 **根本原因分析**

### **为什么MPC在旋转期间会失败？**

**原因1：DR约束过紧**
- DR约束：tracking_error ≤ σ_{k,t} + η_ℰ
- 当v=0时，机器人无法沿参考轨迹前进
- tracking_error累积，超出DR约束边界
- 优化问题变得不可行

**原因2：参考轨迹冲突**
- 参考轨迹继续前进（v>0）
- 但旋转模式要求v=0
- MPC尝试同时满足两个冲突目标
- 求解器无法找到可行解

**原因3：初始猜测问题**
- MPC warm start使用参考轨迹初始化
- 但参考轨迹与旋转要求（v=0）不匹配
- 导致初始点不可行，求解失败

### **原有代码的问题**

**位置**：`src/safe_regret_mpc/src/safe_regret_mpc_node.cpp` 第652-656行

```cpp
} else {
    ROS_WARN("MPC solve failed! Publishing zero velocity...");
    cmd_vel_.linear.x = 0.0;      // ❌ 停止直行
    cmd_vel_.angular.z = 0.0;     // ❌ 停止旋转 ← 问题根源！
}
```

**问题**：
- 当MPC失败时，无条件发布零速度
- **角速度也被设为0**，导致旋转停止
- 机器人必须等待下次MPC成功才能继续旋转
- 但MPC可能持续失败，导致长时间停止

## 🔧 **修复方案**

### **核心思想**

**回退机制**：当MPC失败时，根据当前状态决定回退策略：
- **正常模式**：发布零速度（安全停车）
- **旋转模式**：**继续旋转**（维持角速度）

### **代码实现**

**修改文件**：`src/safe_regret_mpc/src/safe_regret_mpc_node.cpp`

**修改前**（lines 633-656）：
```cpp

    // 获取控制命令
    Eigen::VectorXd control = mpc_solver_->getOptimalControl();
    cmd_vel_.linear.x = control(0);
    cmd_vel_.angular.z = control(1);
    // ... 日志输出 ...
} else {
    // ❌ 无论什么模式，都发布零速度

    cmd_vel_.linear.x = 0.0;

```

**修改后**：
```cpp
if (success) {
    // 获取控制命令
    Eigen::VectorXd control = mpc_solver_->getOptimalControl();
    cmd_vel_.linear.x = control(0);
    cmd_vel_.angular.z = control(1);
    // ... 日志输出 ...
} else {
    // ✅ FIX: 检查是否在旋转模式
    if (mpc_solver_->isInPlaceRotation()) {
        // 旋转模式：继续旋转而不是停止
        double fixed_angular_vel = 0.5;  // rad/s
        double rotation_direction = (etheta >= 0) ? -1.0 : 1.0;
        cmd_vel_.linear.x = 0.0;
        cmd_vel_.angular.z = rotation_direction * fixed_angular_vel;

        ROS_WARN_THROTTLE(2.0, "⚠️  MPC failed but CONTINUING ROTATION: v=%.3f, w=%.3f (etheta=%.1f°)",
                          cmd_vel_.linear.x, cmd_vel_.angular.z, etheta * 180.0 / M_PI);
    } else {
        // 正常模式：安全停车
        ROS_WARN("MPC solve failed! Publishing zero velocity...");
        cmd_vel_.linear.x = 0.0;
        cmd_vel_.angular.z = 0.0;
    }
}
```

### **关键改进**

| 场景 | 修改前 | 修改后 |
|------|--------|--------|
| **正常模式 + MPC失败** | v=0, ω=0 | v=0, ω=0 （安全停车）✅ |
| **旋转模式 + MPC失败** | v=0, ω=0 ❌ | v=0, ω=±0.5 ✅ **继续旋转** |
| **行为** | 转一下停一下 ❌ | 平滑连续旋转 ✅ |

## 🎯 **修复逻辑详解**

### **决策流程**

```
MPC求解失败
    ↓
检查：isInPlaceRotation()?
    ↓
┌─────────────┬─────────────┐
│  正常模式    │  旋转模式    │
│  (false)    │  (true)     │
├─────────────┼─────────────┤
│ 安全停车     │  继续旋转    │
│ v=0, ω=0   │  v=0, ω=±0.5 │
│ ⚠️  停止运动 │  ✅ 保持旋转  │
└─────────────┴─────────────┘
```

### **为什么这样设计？**

**1. 正常模式 - 安全停车**
- MPC失败意味着优化问题不可行
- 可能存在安全隐患（约束冲突）
- 最好的策略：立即停止，避免危险
- 等待下次MPC成功再继续

**2. 旋转模式 - 继续旋转**
- 旋转模式是由角度触发的（>90°）
- **角度不会因为MPC失败而改变**
- 必须继续旋转，直到角度<10°
- 停止旋转会违背状态机的设计目标

**3. 为什么不一直旋转？**
- 正常模式下，MPC失败可能是临时问题
- 如果一直旋转，可能在错误的方向上运动
- 安全停车更符合fail-safe原则
