# 原地旋转接近90度摆头修复报告

**日期**: 2026-04-05
**问题**: 快到90度时左右摆头（MPC角速度振荡）
**状态**: ✅ **已修复**

---

## 🔍 **问题描述**

### **症状**

机器人在接近90度角度误差时（etheta ≈ 85-89°）就开始左右摆头：
- ❌ 此时还未进入原地旋转模式（条件是>90°）
- ❌ MPC的角速度输出不稳定，在正负之间振荡
- ❌ 导致机器人左右摆动，无法稳定过渡到原地旋转

### **时机分析**

| etheta范围 | 模式 | 行为 |
|-----------|------|------|
| 0-60° | 正常 | MPC完全控制 ✅ |
| **60-90°** | **接近区** | **MPC振荡 ❌** |
| >90° | 原地旋转 | 固定角速度 ✅ |

**问题区域**：**60-90°接近区**是摆头高发区！

---

## 🔬 **根本原因分析**

### **MPC在接近90度时的行为**

当etheta接近90度时：
1. **约束冲突**：
   - MPC既要前进（线速度）
   - 又要大角度转向（角速度）
   - 两个目标可能冲突

2. **求解不稳定**：
   - 优化问题接近不可行域边界
   - 小扰动导致解跳变
   - 角速度在正负之间振荡

3. **状态切换延迟**：
   - 在89.5°时：正常模式，MPC控制
   - 在90.5°时：进入旋转模式，固定角速度
   - **突变导致不平稳**

### **原有逻辑的问题**

```cpp
if (abs_etheta > 90°) {
    // 原地旋转模式：固定角速度
} else {
    // 正常模式：MPC完全控制 ← 问题在这里！
}
```

**问题**：
- 60-90°：MPC完全控制，但已经开始振荡
- 90°时突然切换到固定角速度
- **没有平滑过渡**

---

## 🔧 **修复方案：三阶段控制**

### **核心思想**

添加**接近区**（60-90°），在正常模式和原地旋转模式之间平滑过渡。

### **三个控制阶段**

```
┌─────────────┬─────────────┬─────────────┐
│  正常模式    │  接近区     │ 原地旋转    │
│  0-60°      │  60-90°     │  >90°       │
├─────────────┼─────────────┼─────────────┤
│ MPC完全控制 │ 平滑过渡    │ 固定角速度 │
│ 线速度正常 │ 逐渐减速    │ 线速度=0   │
│ 角速度可变 │ 角速度稳定   │ 角速度固定 │
└─────────────┴─────────────┴─────────────┘
```

### **代码实现**

#### **1. 添加接近区阈值**

```cpp
static constexpr double HEADING_ERROR_CRITICAL = 1.57;   ///< 90° - Enter rotation threshold
static constexpr double HEADING_ERROR_APPROACH = 1.05;   ///< 60° - Approach zone start
static constexpr double HEADING_ERROR_EXIT = 0.175;      ///< 10° - Exit rotation threshold
```

#### **2. 三阶段控制逻辑**

```cpp
void SafeRegretMPC::applyRotationSpeedLimit(VectorXd& control, double etheta) {
    double& linear_vel = control(0);
    double& angular_vel = control(1);
    double abs_etheta = std::abs(etheta);

    if (in_place_rotation_) {
        // ✅ 阶段3：原地旋转模式 (>90°)
        linear_vel = 0.0;
        angular_vel = 固定值;

    } else if (abs_etheta > HEADING_ERROR_APPROACH) {
        // ⚠️  阶段2：接近区 (60-90°)
        // 平滑过渡：MPC → 固定值

        // 计算插值因子 (0 at 60°, 1 at 90°)
        double factor = (abs_etheta - 60°) / (90° - 60°);
        factor = std::max(0.0, std::min(1.0, factor));

        // 线速度：逐渐减速
        linear_vel = linear_vel * (1.0 - factor);

        // 角速度：平滑过渡到固定值
        double target_angular_vel = 固定值;
        angular_vel = angular_vel * (1.0 - factor) + target_angular_vel * factor;

    } else {
        // ✅ 阶段1：正常模式 (0-60°)
        // MPC完全控制，不做修改
    }
}
```

### **平滑过渡算法**

#### **插值因子计算**

```cpp
// abs_etheta = 75° 时：
factor = (75 - 60) / (90 - 60) = 15 / 30 = 0.5

// 线速度：
linear_vel = MPC_output * (1 - 0.5) = MPC_output * 0.5  // 减速50%

// 角速度：
angular_vel = MPC_output * 0.5 + fixed_vel * 0.5  // 50% MPC + 50% 固定
```

#### **具体数值示例**

| etheta | factor | 线速度 | 角速度 | 说明 |
|--------|--------|--------|--------|------|
| 60° | 0.0 | 100% MPC | 100% MPC | 开始过渡 ✅ |
| 70° | 0.33 | 67% MPC | 67% MPC + 33% 固定 | 逐渐过渡 |
| 80° | 0.67 | 33% MPC | 33% MPC + 67% 固定 | 大部分固定 |
| 90° | 1.0 | 0% | 100% 固定 | 完全切换 ✅ |

### **1. 提前介入**

| 方面 | 修复前 | 修复后 |
|------|--------|--------|
| **介入时机** | >90°才介入 | >60°开始介入 ✅ |
| **摆头问题** | 60-90°振荡 ❌ | 平稳过渡 ✅ |
| **切换平滑度** | 突变 ❌ | 渐变 ✅ |

### **2. 平滑过渡算法**

```cpp
// 线速度：逐渐减速

// 角速度：加权平均

```

**特点**：
- ✅ 线速度逐渐减小（不突兀）
- ✅ 角速度平滑过渡（避免振荡）
- ✅ factor从0到1线性变化（可预测）
