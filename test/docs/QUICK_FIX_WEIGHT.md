# 🔧 权重过度优化问题修复

## 问题现象
机器人又不会动了

## 根本原因
**mpc_w_etheta权重设置过大（200.0）**，导致MPC过度惩罚航向误差，使得机器人：
- 过度关注转向
- 几乎不前进
- 被困在原地微调方向

## 解决方案
将航向误差权重从200.0降低到80.0：

```yaml
# 修复前
mpc_w_etheta: 200.0  # ❌ 太大，机器人几乎不动

# 修复后
mpc_w_etheta: 80.0   # ✅ 平衡转向和前进
```

## 权重对比

| 权重值 | 效果 | 问题 |
|--------|------|------|
| 50.0 | 原始值 | 可能倒退而非转弯 |
| 80.0 | **当前推荐** | 平衡转向和前进 |
| 200.0 | 过度优化 | 机器人几乎不动 |

## 智能转向逻辑仍然有效

即使降低到80.0，智能转向策略**仍然工作**：

```cpp
// 代码中的硬逻辑（独立于权重）
if(|etheta| > 90°) {
    速度 = 0;  // 纯旋转
} else if(|etheta| > 60°) {
    速度 = 20%; // 转向优先
}
```

这个逻辑不依赖权重，所以大角度时仍会优先转向。

## 如果还有问题

### 机器人仍然太慢
进一步降低权重：
```yaml
mpc_w_etheta: 60.0  # 更少关注航向
```

### 又开始倒退
适度提高权重：
```yaml
mpc_w_etheta: 100.0  # 更多关注航向
```
