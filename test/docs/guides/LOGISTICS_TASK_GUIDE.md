# 物流任务系统使用指南

## 系统概述

新的物流任务系统实现了**取货点 → 卸货点**的自动化导航：

- **取货点 S1 (Pickup Station)**：可变坐标，可通过参数修改
- **卸货点 S2 (Delivery Station)**：固定坐标 (8.0, 0.0)

## 任务流程

```
📍 起始位置: (-8.0, 0.0) - Station S1 (西侧)
    ↓
📦 任务 1: 前往取货点 (默认: 0.0, -7.0)
    ↓
✅ 到达取货点，等待2秒
    ↓
🏁 任务 2: 前往卸货点 (固定: 8.0, 0.0)
    ↓
✅ 到达卸货点，任务完成
```

## 使用方法

### 1. 默认配置（取货点在 (0.0, -7.0)）

```bash
# 清理ROS进程
killall -9 roslaunch rosmaster roscore gzserver gzclient python

# 启动测试（使用默认取货点）
roslaunch safe_regret_mpc safe_regret_mpc_test.launch
```

### 2. 自定义取货点坐标

```bash
# 方式1：通过命令行参数修改取货点
roslaunch safe_regret_mpc safe_regret_mpc_test.launch pickup_x:=3.0 pickup_y:=-7.0

# 方式2：修改取货点为其他位置
roslaunch safe_regret_mpc safe_regret_mpc_test.launch pickup_x:=-5.0 pickup_y:=5.0

# 方式3：同时修改取货点和朝向
roslaunch safe_regret_mpc safe_regret_mpc_test.launch pickup_x:=0.0 pickup_y:=-7.0 pickup_yaw:=1.57
```

### 3. 可用的参数

| 参数 | 默认值 | 说明 | 是否可变 |
|------|--------|------|----------|
| `pickup_x` | 0.0 | 取货点X坐标（米） | ✅ 可变 |
| `pickup_y` | -7.0 | 取货点Y坐标（米） | ✅ 可变 |
| `pickup_yaw` | 0.0 | 取货点朝向（弧度） | ✅ 可变 |
| `delivery_x` | 8.0 | 卸货点X坐标（米） | ❌ 固定 |
| `delivery_y` | 0.0 | 卸货点Y坐标（米） | ❌ 固定 |
| `delivery_yaw` | 0.0 | 卸货点朝向（弧度） | ❌ 固定 |
| `goal_radius` | 0.5 | 目标到达判定半径（米） | ✅ 可调 |

## 参数说明

### 坐标系说明

- 地图范围：20m × 20m
- 坐标范围：X ∈ [-10, 10], Y ∈ [-10, 10]
- 原点 (0, 0)：地图中心

### 关键位置

| 位置 | 坐标 | 说明 |
|------|------|------|
| 机器人起点 | (-8.0, 0.0) | Station S1（西侧） |
| 卸货点 S2 | (8.0, 0.0) | Station S2（东侧，固定） |
| 默认取货点 | (0.0, -7.0) | 南侧中部，可修改 |

### 朝向说明

- `yaw=0.0`：朝向正东（+X方向）
- `yaw=1.57`：朝向正北（+Y方向）
- `yaw=-1.57`：朝向正南（-Y方向）
- `yaw=3.14`：朝向正西（-X方向）

## 自动化测试脚本集成

后续的自动化测试脚本可以通过修改 `pickup_x` 和 `pickup_y` 参数来测试不同的取货点位置：

### 示例：批量测试脚本

```bash
#!/bin/bash
# test_multiple_pickup_points.sh

# 定义多个取货点测试案例
pickup_points=(
    "0.0,-7.0"    # 南侧中部
    "3.0,-7.0"    # 东南侧
    "-3.0,-7.0"   # 西南侧
    "0.0,7.0"     # 北侧中部
    "0.0,0.0"     # 地图中心
)

for point in "${pickup_points[@]}"; do
    IFS=',' read -r x y <<< "$point"

    echo "=========================================="
    echo "测试取货点: ($x, $y)"
    echo "=========================================="

    # 清理之前的进程
    killall -9 roslaunch rosmaster roscore gzserver gzclient python
    sleep 2

    # 启动测试（使用当前取货点）
    roslaunch safe_regret_mpc safe_regret_mpc_test.launch \
        pickup_x:=$x pickup_y:=$y &

    # 等待测试完成（这里需要根据实际情况调整）
    sleep 120

    # 记录结果（可以添加日志收集逻辑）
    echo "完成测试取货点: ($x, $y)"
done
```

## ROS参数接口

其他ROS节点可以通过以下参数获取当前任务配置：

```python
import rospy

# 读取取货点坐标
pickup_x = rospy.get_param('/auto_nav_goal/pickup_x', 0.0)
pickup_y = rospy.get_param('/auto_nav_goal/pickup_y', -7.0)

# 读取卸货点坐标
delivery_x = rospy.get_param('/auto_nav_goal/delivery_x', 8.0)
delivery_y = rospy.get_param('/auto_nav_goal/delivery_y', 0.0)
```

## 注意事项

1. **取货点安全**：确保取货点不在障碍物内，且机器人可以到达
2. **卸货点固定**：卸货点 S2 固定在 (8.0, 0.0)，不建议修改
3. **到达判定**：默认判定半径为 0.5m，可通过 `goal_radius` 参数调整
4. **等待时间**：到达取货点后自动等待 2 秒，再前往卸货点
5. **超时设置**：每个任务的超时时间为 120 秒

## 故障排除

### 问题1：取货点无法到达

**原因**：取货点坐标在障碍物内或地图外

**解决**：
- 检查取货点坐标是否在地图范围内 [-10, 10] × [-10, 10]
- 确保取货点不在货架或墙壁内
- 在RViz中查看costmap确认该位置可达

### 问题2：任务卡住不动

**原因**：路径规划失败或move_base未启动

**解决**：
```bash
# 检查move_base状态
rosnode list | grep move_base
rostopic echo /move_base/GlobalPlanner/plan

# 查看当前目标
rostopic echo /move_base_simple/goal
```

### 问题3：机器人到达目标但未检测到

**原因**：到达判定半径过小

**解决**：增大goal_radius参数
```bash
roslaunch safe_regret_mpc safe_regret_mpc_test.launch \
    pickup_x:=0.0 pickup_y:=-7.0 \
    goal_radius:=1.0  # 增大到1米
```
