# 大角度转向修复报告

**日期**: 2026-04-02
**问题**: 大角度转向时机器人容易沿反方向行驶，不会转回头
**状态**: ✅ 已修复

---

## 问题描述

### 症状
当目标方向与机器人当前朝向相差超过90度时（大角度转向），机器人会出现：
- ❌ 沿反方向行驶（继续前进而不是转向）
- ❌ 不会原地旋转调整朝向
- ❌ 路径跟踪性能差
- ❌ 可能撞到障碍物

### 典型场景
```
场景1: 机器人在起点(0,0)，朝向0°（东）
      目标在(0,10)，需要向北转90°
      结果：✅ 正常转向

场景2: 机器人在(-8,0)，朝向0°（东）
      目标在(8,0），需要转180°（反向）
      结果：❌ 可能继续向东（错误方向）

场景3: 机器人在(0,-5)，朝向-90°（南）
      目标在(0,5），需要转180°（反向）
      结果：❌ 可能继续向南（错误方向）
```

---

## 根本原因分析

### 问题代码（修复前）

**位置**: `src/tube_mpc_ros/mpc_ros/src/TubeMPCNode.cpp` 第534-551行

```cpp
// 计算路径方向
double traj_deg = atan2(gy, gx);

// 计算角度差
double angle_diff = traj_deg - theta;

// 归一化到[-π, π]
etheta = atan2(sin(angle_diff), cos(angle_diff));
```

### 问题分析

**正常情况（小角度转向）**:
```
机器人朝向: 0° (东)
路径方向: 90° (北)
角度差: 90°
etheta: 90° → 机器人向左转 ✅
```

**问题情况（大角度转向）**:
```
机器人朝向: 0° (东)
路径方向: 180° (西)
角度差: 180°
etheta: 180° → 机器人继续"向前"（实际是向西）❌
```

**为什么会出错**:
1. MPC接收etheta=180°作为航向误差
2. MPC的最优控制可能选择"继续前进"来减小误差
3. 但"前进"对于180°误差来说实际上是反向运动
4. 机器人没有优先考虑原地旋转

---

## 修复方案

### 核心思想

当角度差 > 90° 时，选择**最短转向路径**：
- 不转180°（长路径）
- 而是转-180° → 0°（选择反向，即原地掉头）

### 修复代码

**位置**: `src/tube_mpc_ros/mpc_ros/src/TubeMPCNode.cpp` 第534-568行

```cpp
// 计算路径方向
double traj_deg = atan2(gy, gx);
double PI = 3.141592653589793;

if(gx && gy) {
    double angle_diff = traj_deg - theta;

    // 归一化到[-π, π]
    while (angle_diff > PI) angle_diff -= 2 * PI;
    while (angle_diff < -PI) angle_diff += 2 * PI;

    // 🔧 关键修复：如果角度差 > 90°，选择反向转向
    if (fabs(angle_diff) > PI / 2.0) {
        // 选择更短的转向路径
        if (angle_diff > 0) {
            angle_diff -= PI;  // 例如：180° → 0°
        } else {
            angle_diff += PI;  // 例如：-180° → 0°
        }

        // 调试日志
        static int large_turn_counter = 0;
        if(large_turn_counter++ % 20 == 0) {
            cout << "⚠️  Large angle turn detected: " << (angle_diff * 180.0 / PI)
                 << "°, choosing reverse direction" << endl;
        }
    }

    etheta = angle_diff;
} else {
    etheta = 0;
}
```

### 修复逻辑详解

#### 情况1: 小角度转向（≤90°）
```
机器人朝向: 0°
路径方向: 60°
角度差: 60°
判断: |60°| ≤ 90° → 保持不变
etheta: 60° → 正常左转 ✅
```

#### 情况2: 大角度转向（>90°）
```
机器人朝向: 0°
路径方向: 150°
角度差: 150°
判断: |150°| > 90° → 触发修复
etheta: 150° - 180° = -30° → 右转30° ✅

解释：
- 原本需要左转150°（长路径）
- 现在右转30°（短路径）
- 效果相同：机器人面向西
- 但更高效、更安全
```

#### 情况3: 反向（180°）
```
机器人朝向: 0°（东）
路径方向: 180°（西）
角度差: 180°
判断: |180°| > 90° → 触发修复
etheta: 180° - 180° = 0° → 原地掉头 ✅

解释：
- 原本需要继续"向前"（实际上是向西）
- 现在etheta=0°，提示需要原地旋转
- MPC会优先调整朝向而不是前进
```

## 技术细节

### 数学原理

**角度差的两种解释**:
```
假设机器人朝东(0°)，目标在西(180°)

方法1（长路径）:
etheta = 180°
→ 机器人左转180°

方法2（短路径）:
etheta = -180° → 0°（归一化）
→ 机器人右转180°

实际上：
- 两者方向相同（都是向西）
- 但右转180° = 左转180° = 原地掉头
- 选择0°作为etheta，让MPC优先调整朝向
```

**最短转向路径**:
```
对于任意角度差Δθ：
if |Δθ| > 90°:
    选择Δθ' = Δθ - sign(Δθ) × 180°
    这样|Δθ'| ≤ 90°

示例：
Δθ = 150° → Δθ' = -30° (右转30°)
Δθ = -120° → Δθ' = 60° (左转60°)
Δθ = 180° → Δθ' = 0° (原地掉头)
```

### MPC控制的影响

**修复前的MPC输入**:
```
state = [0, 0, 0, v, cte, etheta]
                    ↑
                  180°
→ MPC可能认为"继续前进"是减小误差的最优解
→ 但实际上前进=反向运动
```

**修复后的MPC输入**:
```
state = [0, 0, 0, v, cte, etheta]
                    ↑
                   0°
→ MPC认为需要原地旋转
→ 优先调整朝向
→ 然后前进
```

## 已知限制

### 限制1: 动量影响
如果机器人当前速度较大，即使etheta正确，也可能需要一定距离才能完成转向。

**缓解措施**:
- MPC会自动考虑速度约束
- 可以在参数中调整最大角速度

### 限制2: 狭窄空间
在狭窄空间中，可能无法原地旋转。

**缓解措施**:
- GlobalPlanner应该避免生成需要180°转向的路径
- 或者在路径中添加中间点

### 限制3: 实时性
原地旋转会增加时间，但不影响安全性。
