# 去除最小速度限制修复报告

**日期**: 2026-04-12
**修改**: 允许机器人在低速时更灵活，去除不必要的最小速度限制

---

## 📝 修改内容

### 1. Safe-Regret MPC参数

**文件**: `src/safe_regret_mpc/params/safe_regret_mpc_params.yaml`

#### 输入约束（控制命令）
```yaml
# 修改前
u_min: [0.0, -2.5]      # v_linear最小值=0，不允许倒车

# 修改后
u_min: [-0.3, -2.5]     # v_linear最小值=-0.3 m/s，允许倒车
```

#### 状态约束（速度状态）
```yaml
# 修改前
x_min: [..., 0.0, ...]   # v状态最小值=0
v_min: 0.0               # 终端集速度最小值=0

# 修改后
x_min: [..., -0.3, ...]  # v状态最小值=-0.3 m/s
v_min: -0.3              # 终端集速度最小值=-0.3 m/s
```

---

### 2. Tube MPC参数

**文件**: `src/tube_mpc_ros/mpc_ros/params/tube_mpc_params.yaml`

```yaml
# 修改前
min_trans_vel: 0.1       # 最小移动速度
min_vel_x: -0.1          # 最小线速度
min_rot_vel: 0.1         # 最小旋转速度
trans_stopped_vel: 0.1   # 停止判定阈值
rot_stopped_vel: 0.1     # 旋转停止阈值

# 修改后
min_trans_vel: 0.01      # 降低10倍
min_vel_x: -0.3          # 允许更大倒车速度
min_rot_vel: 0.01        # 降低10倍
trans_stopped_vel: 0.01  # 降低10倍
rot_stopped_vel: 0.01    # 降低10倍
```

---

## 🎯 修改目的

### 问题
1. **最小移动速度过大** (0.1 m/s)：机器人无法低速微调位置
2. **不允许倒车** (min_vel_x = -0.1)：限制机器人在狭窄空间的调整能力
3. **停止阈值过大** (0.1)：机器人可能在目标附近反复震荡

### 解决方案
1. **降低最小速度限制**：0.1 → 0.01 m/s
2. **允许倒车**：min_vel_x = -0.3 m/s
3. **降低停止阈值**：0.1 → 0.01

---

## ⚠️ 注意事项

1. **安全性**：允许倒车需要确保后方无障碍物
2. **性能影响**：更低的速度限制可能导致控制循环频率提高
3. **测试建议**：在开阔区域先测试倒车功能，确认安全后再在复杂环境使用
