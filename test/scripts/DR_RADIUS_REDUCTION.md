# DR Tightening安全半径降低说明

## 📅 修改日期
2026-04-06

## 🎯 修改目标
降低DR Tightening模块的安全半径，减少0.2m

## ✅ 修改的参数

**文件**: `src/dr_tightening/params/dr_tightening_params.yaml`

### 1. Tube Radius（管状半径）
- **原值**: 0.6m
- **新值**: 0.4m
- **降低**: 0.2m
- **说明**: 约束收紧的半径，控制跟踪误差的上限

### 2. Safety Buffer（安全缓冲）
- **原值**: 1.1m
- **新值**: 0.9m
- **降低**: 0.2m
- **说明**: 避障物周围的安全缓冲距离

## 📊 影响分析

### 对约束的影响
- **更紧的约束**: 跟踪误差上限从0.6m降到0.4m
- **更高的灵敏度**: 系统对跟踪误差更敏感
- **可能的影响**:
  - MPC求解可能更容易变得不可行
  - 路径规划可能更加保守

### 对性能的影响
- **优点**: 机器人会更接近障碍物（在某些场景下提高效率）
- **缺点**: 安全裕度降低，可能增加碰撞风险

## 🚀 使用方法

修改后的参数会在下次launch时自动生效：

```bash
roslaunch safe_regret_mpc safe_regret_mpc_test.launch \
    enable_dr:=true \
    debug_mode:=true
```

## ⚠️ 注意事项

1. **安全裕度**: 降低安全半径后，系统安全裕度减少
2. **测试建议**: 在安全环境中先进行测试
3. **监控指标**: 关注MPC可行率和跟踪误差
4. **恢复方法**: 如需恢复原值，将tube_radius改为0.6，safety_buffer改为1.1
