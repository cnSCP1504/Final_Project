### 2. 为什么值恒定（不连续变化）？

**原因**: 路径跟踪非常稳定！

- 机器人沿着规划的全局路径移动
- 到固定目标的距离变化缓慢

- Float32精度（~7位有效数字）导致看起来"恒定"

**实际上**:
- 距离可能在 7.168m 到 7.172m 之间微小变化
- 但robustness都显示为 -6.6684（四舍五入）
- 这是**正常的**，不是bug

**原因**: 到达目标后立即停止监控，没有收集到正值

**时间线**:
- 0-20秒: 前往目标1，robustness ≈ -6.67 (distance > 0.5m)

- 20-100秒: 前往目标2，robustness ≈ -15.49 (distance > 0.5m)

  - 此时robustness应该 = 0.5 - 0.478 = **0.022 > 0** ✅

- ✅ 机器人**确实到达了目标** (distance < 0.5m)
- ✅ 到达时robustness **应该 > 0**
- ❌ 但数据收集在到达时停止，所以没有记录到正值

goal_pose = self.global_path.poses[-1].pose

## 📚 技术要点

### STL Robustness计算
```
ρ(φ; x) = threshold - distance_to_goal
```

其中：
- `threshold` = 0.5m (到达阈值)
- `distance_to_goal` = 当前位置到目标的距离
- `ρ ≥ 0` → 满足STL（到达目标）
- `ρ < 0` → 违反STL（未到达目标）

### 为什么"只有两种值"是正常的？
1. 每个测试有2个目标（取货 + 卸货）
2. 每个目标对应一个固定的robustness值
3. 路径跟踪稳定，距离变化缓慢
4. 所以robustness值看起来是"恒定"的
