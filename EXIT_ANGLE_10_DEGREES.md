# 原地旋转退出角度调整：30度 → 10度

**日期**: 2026-04-03
**修改**: 将强制转弯的退出角度从30度降低到10度
**原因**: 提高转弯精度，让机器人更精确地对齐路径

---

## 修改的参数

### 参数：HEADING_ERROR_EXIT

**文件**: `src/tube_mpc_ros/mpc_ros/src/TubeMPCNode.cpp`

**修改内容**:
```cpp
// 旧值
const double HEADING_ERROR_EXIT = 0.524;  // 30度

// 新值
const double HEADING_ERROR_EXIT = 0.175;  // 10度
```

---

## 行为变化对比

### 旧版本（30度退出）

| 角度范围 | 状态 | 速度 | 说明 |
|---------|------|------|------|
| > 90° | 进入旋转 | 0.0 m/s | 开始原地旋转 |
| 30-90° | 保持旋转 | 0.0 m/s | 继续旋转 |
| < 30° | 退出旋转 | 恢复正常 | **开始直行** |

**问题**：在25度时就开始直行，可能还没完全对齐路径

### 新版本（10度退出）

| 角度范围 | 状态 | 速度 | 说明 |
|---------|------|------|------|
| > 90° | 进入旋转 | 0.0 m/s | 开始原地旋转 |
| 10-90° | 保持旋转 | 0.0 m/s | 继续旋转 |
| < 10° | 退出旋转 | 恢复正常 | **开始直行** |

**改进**：在5度时才退出，路径对齐更精确

## 预期效果

### ✅ 优点

1. **更精确的路径对齐**：
   - 机器人会旋转到更接近目标方向（<10度）
   - 直行时与路径偏差更小

2. **减少路径偏离**：
   - 旧版本：可能在25度就开始直行，导致后续需要频繁调整
   - 新版本：在10度以下才直行，后续调整更少

3. **更好的跟踪性能**：
   - CTE（cross-track error）和etheta（heading error）都会降低

### ⚠️ 可能的副作用

1. **旋转时间增加**：
   - 从90度旋转到10度需要更多时间
   - 可能延长整体导航时间

2. **在大角度连续转弯场景**：
   - 如果路径频繁变化，可能导致长时间保持在旋转模式
   - 影响整体进度
